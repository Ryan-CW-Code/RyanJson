# RyanJson 项目导航工作流（压缩版）

## 1. 先定用户阶段
1. 第一次接触仓库。
2. 已经开始接入或使用 API。
3. 正在改内部实现或排查 crash。
4. 正在补测试或做回归治理。

## 2. 首屏回答顺序
1. 一句话解释 RyanJson 仓库是什么。
2. 给 3 到 5 个第一跳入口。
3. 指出当前问题更适合的 skill、脚本或报告。

## 3. 问题模糊时
1. 先看 `../shared/questionRouter.md`。
2. 再用 `references/entryScenarios.md` 选最贴近的入门场景。
3. 默认给最短阅读路径，不展开细节。

## 4. 常见漏项
- 只列目录，不告诉用户先读顺序。
- 只给脚本名，不说适用场景。
- 没把导航问题继续分流到 API/优化/测试 skill。

## 5. 依据（仓库内）
- `references/repoMap.md`
- `references/buildAndValidation.md`
- `references/docsAndReports.md`
- `references/entryScenarios.md`
